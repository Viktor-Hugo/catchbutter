# CatchButter

길드원끼리 PC와 모바일 브라우저에서 바로 접속해 즐길 수 있는 로컬 우선 그림 맞히기 게임 프로토타입입니다.

## 배포

현재 구조는 서버가 빌드된 프론트를 같이 서빙하는 단일 Node 서비스입니다. 그래서 Railway 같은 Node 호스팅에 바로 올릴 수 있습니다.

### Railway 기준

1. GitHub에 이 프로젝트를 올립니다.
2. Railway에서 `New Project` -> `Deploy from GitHub repo`를 선택합니다.
3. 저장소를 연결합니다.
4. 루트 폴더 그대로 배포합니다.

이 저장소에는 이미 [`railway.json`](./railway.json)이 들어 있어서, Railway가 아래 값을 그대로 사용합니다.

```txt
Build Command: npm install && npm run build
Start Command: npm run start
Health Check Path: /health
```

5. 배포가 끝나면 발급된 `https://...up.railway.app` 주소를 길드원에게 공유합니다.

주의:

- Socket.IO를 쓰므로 반드시 정적 사이트가 아니라 Node 서비스로 배포해야 합니다.
- Railway 요금 정책은 바뀔 수 있으니, 실제 배포 전에 현재 플랜 조건은 본인 계정 기준으로 한 번 확인하는 편이 안전합니다.
- 처음 배포 전에는 로컬에서 `npm install`, `npm run build`, `npm run start` 가 모두 통과하는지만 보면 충분합니다.

## 구성

- `client`: React + Vite 기반 모바일 우선 UI
- `server`: Express + Socket.IO 기반 실시간 게임 서버
- `npm run dev`: 프론트와 서버를 동시에 실행
- `npm run build && npm run start`: 빌드된 프론트를 서버가 직접 서빙

## 빠른 시작

### 1. 개발 모드

```bash
npm run dev
```

- 클라이언트: `http://localhost:5173`
- 서버: `http://localhost:3001`

같은 와이파이에서 테스트하려면 PC의 내부 IP로 접속하면 됩니다.

예시:

- 클라이언트: `http://192.168.0.10:5173`
- 서버 소켓: 자동으로 `http://192.168.0.10:3001` 로 연결

### 2. 실제 플레이용 단일 서버 모드

```bash
npm run build
npm run start
```

이 모드에서는 서버가 `client/dist`를 같이 서빙하므로 한 주소만 열면 됩니다.

- 게임 주소 예시: `http://localhost:3001`

주의:

- 콘솔에 `0.0.0.0:3001` 이 보이더라도 그 주소를 브라우저에 넣는 것은 아닙니다.
- 같은 PC에서는 `http://localhost:3001` 또는 `http://127.0.0.1:3001` 로 접속합니다.
- 같은 와이파이의 다른 기기에서는 내 PC의 내부 IP 예: `http://192.168.0.10:3001` 로 접속합니다.

터널링은 이 모드에 붙이는 편이 가장 단순합니다.

## 게임 규칙

- 첫 입장자가 호스트가 됩니다.
- 2명 이상 있어야 시작할 수 있습니다.
- 총 6라운드 동안 그리는 사람이 순서대로 바뀝니다.
- 출제자는 제시어를 보고 시간 제한 없이 그림을 그리고, 나머지는 채팅창으로 정답을 맞힙니다.
- 누군가 먼저 맞혀도 그림은 계속 이어지고, 정답을 맞힌 사람은 점수를 얻습니다.
- 출제자는 원할 때 직접 다음 라운드로 넘길 수 있습니다.
- 출제자를 제외한 모든 인원이 정답을 맞히면 자동으로 다음 라운드로 넘어갑니다.
- 아직 모두가 못 맞힌 상황에서 빨리 넘기고 싶으면 출제자를 제외한 모든 인원이 `넘기기`에 동의하면 됩니다.

## 터널링 운영 방식

추천 흐름은 아래처럼 잡으면 됩니다.

1. `npm run build`
2. `npm run start`
3. 터널링 도구로 `3001` 포트를 외부에 노출
4. 생성된 링크를 길드원에게 공유

중요한 점:

- 길드원은 별도 설치 없이 모바일 브라우저로 접속하면 됩니다.
- 서버는 계속 내 PC에서 실행 중이어야 합니다.
- 무료 터널은 주소가 매번 바뀔 수 있습니다.

## 배포용 체크포인트

- `npm install` 한 번으로 루트, 서버, 클라이언트 의존성이 같이 설치됩니다.
- `npm run build` 는 `client/dist`를 만듭니다.
- `npm run start` 는 Express 서버가 정적 파일과 Socket.IO를 함께 서빙합니다.
- 배포 환경에서는 `PORT` 환경변수를 플랫폼이 자동 주입해도 그대로 동작합니다.
